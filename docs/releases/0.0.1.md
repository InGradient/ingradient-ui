# Release 0.0.1

Release date: 2026-03-20

## Summary

`0.0.1`은 현재 `ingradient-ui`의 첫 기준 버전이다. 루트 패키지 구조, token source 전략, 사용자용 docs app, 유지보수 문서 체계를 한 번에 고정한 초기 릴리즈로 본다.

## Included

- repo 루트가 곧 `@ingradient/ui` 패키지인 구조
- `src/` source of truth, `lib/` generated output 구조
- `tokens -> primitives -> components -> patterns -> brand` 공용 UI 레이어
- TypeScript token source + generated `lib/tokens.css`
- `@ingradient/ui/legacy` 호환 subpath
- `apps/design-showcase` 기반 사용자용 docs app
- `docs/` 기반 유지보수 문서 체계

## Not Included

- dataset, annotation, export workflow
- camera, device, capture workflow
- project/member/class/training/model business logic
- 앱별 router, API, permission logic

이런 것은 여전히 소비 앱 저장소가 소유한다.

## Highlights

### Package Structure

- `packages/ui` 구조를 제거하고 루트 패키지 구조로 정리했다.
- public export는 `@ingradient/ui`, `@ingradient/ui/tokens`, `@ingradient/ui/primitives`, `@ingradient/ui/components`, `@ingradient/ui/patterns`, `@ingradient/ui/brand`, `@ingradient/ui/legacy`로 고정했다.

### Styling System

- token source를 TypeScript로 고정했다.
- semantic token과 generated CSS variable contract를 연결했다.
- 빌드 시 `lib/tokens.css`가 자동 생성되도록 만들었다.

### Documentation

- showcase를 단순 demo 앱이 아니라 사용자용 API 문서 앱으로 바꿨다.
- 상세 페이지는 `What it is`, `When to use`, `Import`, `Live examples`, `Variants`, `Props`, `Related`, `Do / Don’t`, `Notes` 구조를 따른다.
- 운영 문서는 `docs/`에서 별도 관리하도록 분리했다.

## Known Constraints

- 현재 문서 메타는 자동 추출이 아니라 수동 관리다.
- showcase는 public API를 사용자 관점에서 묶어 문서화한다. 내부 helper나 low-level alias를 모두 1:1 페이지로 노출하지는 않는다.
- chart chunk가 상대적으로 크다. 현재는 수용 가능한 범위지만 추후 code splitting 전략을 계속 조정할 수 있다.

## Upgrade Guidance

이 버전부터는 아래 원칙을 따른다.

- 새 코드는 `@ingradient/ui/legacy` 대신 범용 public API를 우선 사용한다.
- 토큰 수정은 `src/tokens/**`에서 하고, `lib/tokens.css`는 직접 수정하지 않는다.
- public export를 바꾸면 showcase metadata와 `docs/`도 같이 수정한다.
