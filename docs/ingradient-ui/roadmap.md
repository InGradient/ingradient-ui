# ingradient-ui Roadmap

## 현재 범위

- design tokens
- generic components
- reusable patterns
- brand assets
- docs showcase

## 단계별 계획

### 1. 공용 패턴 계약 정리

- layout ownership
- overlay layering
- responsive behavior

### 2. component extraction and quality

- generic reusable pieces 정리
- docs coverage 강화
- style literal 제거

### 3. release and consumption 안정화

- publish flow 정리
- consuming app migration 지원
- boundary 문서 강화

## 우선순위

1. 재사용성 높은 패턴 품질
2. public contract와 docs 정합성
3. package release 안정화

## 기술 부채

- 일부 layout, overlay, brand 영역의 contract가 약하다
- data-heavy product 요구를 충분히 일반화하지 못한 부분이 있다

## 미결정 사항

- 일부 pattern의 공용화 범위
- publish cadence와 semver 운영 방식
